<kol-heading _level="1">Get Started</kol-heading>
<kol-heading _level="2">Verwendung auf statischen Webseiten</kol-heading>

<p>
	<b>KoliBri</b> kann ähnlich einfach und komfortabel wie andere JavaScript-Bibliothekne, z.B. <b>jQuery</b>, auf statischen Webseiten eingebunden und verwendet
	werden.
	<br />
	Aufgrund des Renderings in Shadow-Doms ist die Nutzung der Input-Komponenten auf statischen Webseiten eingeschränkt.
</p>
<kol-heading _level="3">Schritt für Schritt-Anleitung</kol-heading>
<p>
	<b>1. Herunterladen der Bibliotheksordner</b>
</p>
<p>
	Laden Sie die Bibliotheksordner herunter:
	<kol-link _href="kolibri.tgz">Download (tgz)</kol-link>
</p>
<p>
	<b>2. Entpacken Sie den Bibliotheksordner</b>
</p>
<p>
	Im Bibliotheksordner sind sowohl die benötigten Komponenten, wie auch eine Demo-Applikation und das offizielle Storybook enthalten. Für die Nutzung von
	KoliBri auf statischen Webseiten werden diese Inhalte nicht alle benötigt.
	<br />
	Nach dem Entpacken stehen Ihnen folgende Ordner und Dateien zur Verfügung:
</p>
<ul>
	<li>
		Ordner <b>demo</b> : In diesem Ordner befinden sich Dateien, die Sie für die Verwendung auf statischen Webseiten benötigen.
	</li>
	<li>
		Ordner <b>demo@temp</b>: Dieser Ordner wird für die Verwendung auf statischen Webseiten nicht benötigt.
	</li>
	<li>
		Datei <b>README.md</b>: Hier finden Sie weitere Informationen zur Installation bzw. Einbindung von KoliBri in Ihr Projekt.
	</li>
	<li>
		Ordner <b>storybook</b>: Dieser Ordner wird für die Verwendung auf statischen Webseiten nicht benötigt.
	</li>
</ul>
<p>
	<b>3. Einfügen in Ihr Projekt</b>
</p>
<p>
	Nachfolgend wird davon ausgegangen, dass sich Ihr Projekt auf einem Webserver befindet. Im nächsten Schritt fügen Sie alle erforderlichen Systemkomponenten
	von KoliBri in das Root-Verzeichnis ein.
</p>
<p>
	<ol>
		<li>
			Im Ordner <b>demo</b> finden Sie eine Datei <b>demo/kolibri-lib.tgz</b>. Entpacken Sie diese. Sie erhalten einen Ordner <b>package</b>.
		</li>
		<li>
			Verschieben Sie den Ordner <b>package/dist/kolibri</b> in das Root-Verzeichnis Ihres Projektes. Damit stehen Ihnen alle Ressourcen zur Verfügung, die zum
			Betrieb von KoliBri erforderlich sind.
		</li>
		<li>
			Fügen Sie in den <code>head</code> Ihrer HTML-Datei folgende Links ein:
		</li>
	</ol>
</p>

```html
<link rel="stylesheet" href="kolibri/assets/bund/bund.css" />
<link rel="stylesheet" href="kolibri/assets/icofont/icofont.css" />
<link rel="stylesheet" href="kolibri/assets/themes/default.css" />
<link rel="stylesheet" href="kolibri/assets/style.css" />
<script type="module" src="kolibri/kolibri.esm.js"></script>
```

<p>Sie können KoliBri nun nutzen. Achten Sie darauf, das auf statischen Webseiten die Syntax für Web Components erforderlich ist.</p>

<kol-heading _level="2">Beispiel: Aufbau einer statischen Webseite</kol-heading>

```html
<DOCTYPE html>
	<html>
		<head>
			<title>KoliBri | Eine barrierefreie Komponenten-Bibliothek</title>
			<meta name="viewport" content="width=device-width, initial-scale=1" />
			<meta charset="UTF-8" />
			<meta name="kolibri" content="dev-mode=false" />
			<link rel="stylesheet" href="kolibri/assets/bund/bund.css" />
			<link rel="stylesheet" href="kolibri/assets/icofont/icofont.css" />
			<link rel="stylesheet" href="kolibri/assets/themes/default.css" />
			<link rel="stylesheet" href="kolibri/assets/style.css" />
			<script type="module" src="kolibri/kolibri.esm.js"></script>
		</head>
		<body class="default">
			<kol-heading _level="1">Willkommen bei KoliBri!</kol-heading>
		</body>
	</html>
</DOCTYPE>
```
